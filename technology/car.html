Community Robot Car

Build and program a robot car using the ELEGOO Smart Car Kit. Your car will serve your community — as an ambulance, a bus, a plow truck, or whatever your town needs.

First: What Service Does Your Car Provide?

Every vehicle in a community has a job. Decide with your partner what your robot car does for the town. This will change how you program it.

Ambulance

Flashing lights, siren sounds, fast response. Rushes to the hospital house when there's an emergency.

Snow Plow

Slow and steady. Clears the roads for other vehicles. Reliable, never stops.

Bus

Stop-and-go route. Picks up and drops off passengers at each building in town.

Delivery Bot

Carries supplies between buildings. Navigates a route and returns to base.

Police Car

Patrols the streets with lights. Responds to alerts from community buildings.

Your Idea!

Fire truck? Taxi? Ice cream truck? If you can explain how it helps, build it.

Day 5 — Build the Robot Base

Every robot starts as a pile of parts. Today you'll assemble the chassis (that's the frame and body), mount the motors, and wire up the motor driver board.

Today's Goal

Follow the kit instructions to build the car chassis. Mount the 2 DC motors and connect them to the L298N motor driver. Make sure everything is secure — a loose wire will cause headaches later.

What Is a Motor Driver?

The Arduino board is the brain, but it can't power motors directly — they need too much electricity. The L298N motor driver is like a middleman: the Arduino tells it what to do, and it sends the right power to the motors. Four control pins (IN1-IN4) tell each motor which direction to spin.

Pin Connections (Write These Down!)

IN1 = Pin 8, IN2 = Pin 9 (Left Motor) / IN3 = Pin 10, IN4 = Pin 11 (Right Motor). Your kit may differ — check the included guide and update the pin numbers in your code if needed.

Day 6 — Movement: Making It Drive

Time to make your car move. You'll start with the basics — forward, backward, turning — then program movement patterns that match your vehicle's job.

Starter Code: Forward and Stop

Upload this to see your car drive forward for 2 seconds, then stop for 2 seconds, on repeat.

int IN1 = 8; int IN2 = 9; int IN3 = 10; int IN4 = 11; void setup() { pinMode(IN1, OUTPUT); pinMode(IN2, OUTPUT); pinMode(IN3, OUTPUT); pinMode(IN4, OUTPUT); } void loop() { // Forward: both motors spin the same direction digitalWrite(IN1, HIGH); digitalWrite(IN2, LOW); digitalWrite(IN3, HIGH); digitalWrite(IN4, LOW); delay(2000); // Stop: all pins LOW digitalWrite(IN1, LOW); digitalWrite(IN2, LOW); digitalWrite(IN3, LOW); digitalWrite(IN4, LOW); delay(2000); }

How It Works

Each motor has two control pins. Setting one HIGH and the other LOW makes it spin one direction. Swapping them makes it spin the other way. Setting both LOW stops it. That's it — forward, backward, and stop are just different combinations of HIGH and LOW.

Turning

To turn, you run only one motor. The car pivots around the stopped wheel.

// Turn Right: left motor on, right motor off digitalWrite(IN1, HIGH); digitalWrite(IN2, LOW); digitalWrite(IN3, LOW); digitalWrite(IN4, LOW); // Turn Left: right motor on, left motor off digitalWrite(IN1, LOW); digitalWrite(IN2, LOW); digitalWrite(IN3, HIGH); digitalWrite(IN4, LOW);

Pick Your Challenge Level

Basic

Get the car moving forward and stopping. Try changing the delay numbers to control how far it goes.

Medium

Program a full route: forward, turn right, forward, turn left. Make the car drive in a square.

Advanced

Create reusable functions (see below) and program a complete route through your community.

Day 7 — Features: Give Your Car a Personality

A robot that just drives is boring. Today you'll add lights, sounds, and behavior patterns that match your vehicle's role in the community.

Clean Code With Functions

Instead of repeating the same pin commands over and over, let's create named functions. This makes your code easier to read and modify.

int IN1 = 8; int IN2 = 9; int IN3 = 10; int IN4 = 11; void setup() { pinMode(IN1, OUTPUT); pinMode(IN2, OUTPUT); pinMode(IN3, OUTPUT); pinMode(IN4, OUTPUT); } void moveForward() { digitalWrite(IN1, HIGH); digitalWrite(IN2, LOW); digitalWrite(IN3, HIGH); digitalWrite(IN4, LOW); } void moveBackward() { digitalWrite(IN1, LOW); digitalWrite(IN2, HIGH); digitalWrite(IN3, LOW); digitalWrite(IN4, HIGH); } void turnRight() { digitalWrite(IN1, HIGH); digitalWrite(IN2, LOW); digitalWrite(IN3, LOW); digitalWrite(IN4, LOW); } void turnLeft() { digitalWrite(IN1, LOW); digitalWrite(IN2, LOW); digitalWrite(IN3, HIGH); digitalWrite(IN4, LOW); } void stopCar() { digitalWrite(IN1, LOW); digitalWrite(IN2, LOW); digitalWrite(IN3, LOW); digitalWrite(IN4, LOW); } void loop() { moveForward(); delay(2000); turnRight(); delay(800); moveForward(); delay(2000); stopCar(); delay(1000); }

What Are Functions?

A function is a named chunk of code you can reuse. Instead of writing all 4 digitalWrite lines every time, you just say moveForward(). Think of it like naming a play in a sport — one word and everyone knows what to do.

Vehicle Behavior Recipes

Pick the recipe that matches your car's role, then modify it to make it your own.

Ambulance Mode

Fast movement with flashing emergency lights.

int ledPin = 7; void setup() { // ... motor pins setup ... pinMode(ledPin, OUTPUT); } void ambulanceRun() { moveForward(); // Flash lights while driving digitalWrite(ledPin, HIGH); delay(200); digitalWrite(ledPin, LOW); delay(200); digitalWrite(ledPin, HIGH); delay(200); digitalWrite(ledPin, LOW); delay(200); }

Bus Route Mode

Stop-and-go pattern. Drives to a stop, waits, then continues.

void busRoute() { // Drive to stop 1 moveForward(); delay(2000); // Stop for passengers stopCar(); delay(3000); // Drive to stop 2 moveForward(); delay(2000); // Turn onto next street turnRight(); delay(800); // Stop for passengers stopCar(); delay(3000); }

Snow Plow Mode

Slow, steady back-and-forth sweeping pattern.

void snowPlow() { // Slow steady forward sweep moveForward(); delay(3000); // Turn around turnRight(); delay(1500); // Sweep back moveForward(); delay(3000); // Turn around for next pass turnLeft(); delay(1500); }

Add Sound (If Your Kit Has a Buzzer)

Some kits include a buzzer. If yours does, you can add sirens, horns, or jingles.

int buzzer = 6; void setup() { pinMode(buzzer, OUTPUT); } // Simple siren void siren() { tone(buzzer, 1000); // high pitch delay(300); tone(buzzer, 600); // low pitch delay(300); noTone(buzzer); } // Quick honk void honk() { tone(buzzer, 800); delay(150); noTone(buzzer); }

Day 8 — Community Showcase

This is it — the day your community comes together. All 3 pairs in your group will combine their projects into one working town. Houses and cars, all connected by the story of your community.

Setup

Arrange your community's 3 houses on the table. Mark out roads between them with tape. Position your 3 cars at their starting points. This is your town.

Tell the Story

Each community will present their town. Be ready to answer: What is your town called? What does each building do? What services do the cars provide? How do they work together? Show us a demo!

Connection Ideas

Can a house's emergency light signal that the ambulance car should respond? Can the bus visit each building in order? Can the plow clear a path before the delivery bot drives through? Even if the connection is acted out rather than coded, it shows how technology works as a system.
