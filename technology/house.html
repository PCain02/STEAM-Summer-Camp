Build a smart house for your community using the ESP32 IoT kit. Learn how sensors, lights, and automation work together — then customize your house to serve your town.

Day 1 — What Is IoT?

IoT stands for the Internet of Things. It means everyday objects — lights, doors, thermostats — that can sense their environment and react automatically. Your smart house will do exactly that.

Today's Goal

Day 2 — LEDs: Lights That Communicate

Lights aren't just for seeing in the dark — they send messages. A red light means stop. A flashing light means emergency. Today you'll wire up LEDs and make them communicate something for your building.

Starter Code: Blinking LED

Upload this to your board. It makes an LED blink on and off every second.

int ledPin = 7; void setup() { pinMode(ledPin, OUTPUT); } void loop() { digitalWrite(ledPin, HIGH); delay(1000); digitalWrite(ledPin, LOW); delay(1000); }

How It Works

pinMode tells the board that pin 7 is an output (it sends power out). digitalWrite HIGH turns the LED on. LOW turns it off. delay(1000) waits 1000 milliseconds (1 second) before the next step.

Pick Your Challenge Level

Basic

Get the LED blinking. Change the delay numbers to make it blink faster or slower.

Medium

Wire up 2 or 3 LEDs. Make them blink in a pattern — like a traffic signal or emergency strobe.

Advanced

Create a signal system for your building. What do different light patterns mean? Code at least 3 different signals.

Community Connection

What do your lights tell the town? A hospital might flash red for emergencies. A fire station might use alternating red and white. A community center might glow a welcoming steady color. Decide what your lights mean.

Day 3 — Sensors: Your House Reacts

Smart buildings don't just sit there — they respond to the world around them. Today you'll add a button and a light sensor so your house can detect things and react.

Code A: Button Controls Light

Press the button, the light turns on. Let go, it turns off. Simple, but powerful.

int ledPin = 7; int buttonPin = 2; void setup() { pinMode(ledPin, OUTPUT); pinMode(buttonPin, INPUT); } void loop() { int buttonState = digitalRead(buttonPin); if (buttonState == HIGH) { digitalWrite(ledPin, HIGH); } else { digitalWrite(ledPin, LOW); } }

How It Works

digitalRead checks whether the button is being pressed (HIGH) or not (LOW). The if statement is a rule: if the button is pressed, turn the light on. This is the foundation of all smart devices — if something happens, then do something.

Code B: Auto Lights (Light Sensor)

This one is magic — the house turns its own lights on when it gets dark.

int lightPin = A0; int ledPin = 7; void setup() { pinMode(ledPin, OUTPUT); Serial.begin(9600); } void loop() { int lightValue = analogRead(lightPin); Serial.println(lightValue); if (lightValue < 500) { // It's dark — turn the light on digitalWrite(ledPin, HIGH); } else { // It's bright — light off digitalWrite(ledPin, LOW); } delay(500); }

How It Works

analogRead reads the light sensor and gives a number. Low numbers mean it's dark. High numbers mean it's bright. We set a threshold at 500 — you can change that number to make your house more or less sensitive. Open the Serial Monitor to see the live numbers!

Pick Your Challenge Level

Basic

Get the button controlling the LED. Think of it as a doorbell for your building.

Medium

Get the auto-light working. Adjust the threshold so it triggers at the right time. Cover the sensor with your hand to test it.

Advanced

Combine both! Button AND light sensor controlling lights. Can you make the button override the automatic system?

Day 4 — Automation: Smart Rules

Real smart buildings follow rules. "If it gets dark, turn on lights." "If someone presses the emergency button, flash the alarm." Today you'll combine everything into a rule-based system — just like a real IoT building.

Smart Rule Code

This code combines the light sensor AND the button. The house reacts to either one.

int lightPin = A0; int buttonPin = 2; int ledPin = 7; void setup() { pinMode(ledPin, OUTPUT); pinMode(buttonPin, INPUT); } void loop() { int lightValue = analogRead(lightPin); int buttonState = digitalRead(buttonPin); // Rule: Turn on lights if it's dark OR emergency button is pressed if (buttonState == HIGH || lightValue < 400) { digitalWrite(ledPin, HIGH); } else { digitalWrite(ledPin, LOW); } }

The Key Idea

The || symbol means OR. So the rule says: "If the button is pressed OR it's dark, turn the light on." You can also use && which means AND — that would mean BOTH conditions need to be true. Technology follows rules, just like a community does.

Write Your Smart Laws

Every building in your community needs rules. Write them in plain English first, then turn them into code.

Example Rules

"If it's nighttime, turn on the porch light." / "If emergency button is pressed, flash all lights." / "If it's bright out AND no one's pressed the button, stay dark and save power."
